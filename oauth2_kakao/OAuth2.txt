1.2.단계는 인증요청이 가능한 애플리케이션인지 검증하고 허가하는 과정
3.4.단계는 허가(grant) 코드를 통해서 액세스 토큰을 발급하는 과정
5.6.단계는 액세스 토큰을 가지고 필요한 서비스에 접근
-> 1~4단계가 완료되면 액세스 토큰 지속시간동안 5.6단계의
   과정으로 필요한 서비스에 접근 가능함

1단계. 애플리케이션에서 인증서비스 사업자로 인증 허가 요청
  -> 인증서비스 사업자에 애플리케이션이 등록되어있으면
     인증서비스 사업자의 로그인 페이지로 이동됨

2단계. 인증서비스 사업자가 인증 요청을 허가함(허가 코드를 줌)
  -> 사용자 로그인을 하면 사업자 사전에 등록되어 있는 
     리다이렉트 페이지로 허가(grant) 코드를 포함하여 이동시킴
     : 리다이렉트 페이지는 허가 코드를 받는 애플리케이션의 페이지
  -> 코드를 가지고 인증요청을 할 수 있음
